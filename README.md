# Проект. Проверка гипотезы 6 — Как лучше моделировать сезонность временных рядов

Проект исследует, как лучше всего моделировать сезонность временных рядов на данных M4 Weekly.

## Структура

```
config.py - параметры запуска (пути, горизонт, признаки и др.)
main.py — точка входа, запускает все эксперименты
src/
    data.py — загрузка и подготовка данных (перевод в нужный формат строки-столбцы)
    models.py — модели: CatBoostRecursive, StatsforecastModel, BaseModel
    feature_generation.py — генерация признаков: лаги, сезонные лаги, календарь, Фурье
    index_slicing.py — индексация окон для обучения и прогноза
    validation.py — expanding window валидация
    experiments.py — запуск экспериментов с разными конфигурациями признаков и бейзлайнами
    metrics.py — SMAPE, сила сезонности (STL), таблицы метрик
    visualisations.py — графики сравнения моделей, сезонности и горизонта прогнозирования
data/
    m4_weekly_train.csv — обучающая выборка M4 Weekly
plots/ — результаты: графики (.png) и таблица метрик (.csv)
```

## Зависимости

Зависимости зафиксированы в `pyproject.toml` и `uv.lock`.

## Запуск

Настройте параметры в `config.py` (ряды, горизонт, директория для графиков), затем запуститесь из main.py

Результаты сохранятся в директорию, указанную в `OUTPUT_DIR`.
